2.Difference among local storage, session storage and cookies

Local storage Session storage Cookies
Data stored in local storage persists even after the browser is closed and reopened. It has no expiration date by default and remains available until explicitly removed by the web application or the user. Data stored in session storage is only available for the duration of a single page session. When the user closes the browser tab or window, the session storage data is automatically cleared. JavaScript cookies are small pieces of data that can be stored in a user's web browser. They are commonly used for various purposes in web development, such as remembering user preferences, tracking user behavior, and maintaining session information.
Local storage typically has a larger storage limit compared to session storage. Browsers often allow at least 5-10 MB of storage space per domain for local storage. Session storage typically has a smaller storage limit compared to local storage. Browsers usually provide around 5-10 MB of storage space per domain for session storage. Cookies are typically limited to a few kilobytes in size and can store small amounts of data, such as strings. This data is stored on the user's local machine.

3.What is event loop? How does work it?

Event Loop: An event loop is something that pulls stuff out of the queue and places it onto the function execution stack whenever the function stack becomes empty.

Working Process: An event loop is an endless loop, which waits for tasks, executes them, and then sleeps until it receives more tasks. The event loop executes tasks from the event queue only when the call stack is empty i.e. there is no ongoing task. The event loop allows us to use callbacks and promises.
